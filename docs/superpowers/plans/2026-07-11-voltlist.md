# VoltList Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. This plan is executed by a **Workflow orchestrating parallel agents** (see §Execution).

**Goal:** A full-stack web app cataloging every EV/PHEV/HEV sold new in the USA (MY2024–2026) with complete, sourced specs, faceted filtering, account-synced favorites & cart, side-by-side compare, and CSV/PDF export — shipped as one GHCR Docker image with a compose file.

**Architecture:** npm-workspaces monorepo: `shared` (Zod vehicle schema + DTO types), `server` (Express + Prisma/SQLite + JWT auth; cars served from an in-memory store loaded from validated `data/cars.json`; SQLite holds only User/Favorite/CartItem), `web` (React 18 + Vite + TS + Tailwind SPA). Production server serves the built SPA and the API from one process. Dataset is curated by parallel research agents and validated by a build-failing test.

**Tech Stack:** Node 20, TypeScript, Express 4, Prisma 5 + SQLite, bcryptjs, jsonwebtoken, cookie-parser, helmet, express-rate-limit, zod, pdfkit; React 18, Vite 5, Tailwind 3, @tanstack/react-query, react-router-dom; Vitest + Supertest + @testing-library/react; Docker (multi-stage) + GitHub Actions → GHCR.

## Global Constraints

- Node 20; TypeScript strict mode everywhere.
- All packages public; image `ghcr.io/atvriders/voltlist` public.
- Model years covered: **2024, 2025, 2026** only.
- Powertrain types: `BEV | PHEV | HEV` only.
- No external image hotlinking; styled spec cards only (CSP forbids remote images).
- Every vehicle field carries provenance; each vehicle has `sources[]` and `dataAsOf`.
- One commit at the very end, after full verification (typecheck + both builds + full test suite green). Push only on explicit user request.
- Money in whole USD integers; distances in miles; energy in kWh; all enums exactly as defined in `shared`.

## Shared Contracts (all workstreams consume these verbatim)

### `shared/src/vehicle.ts` — the canonical schema

```ts
import { z } from "zod";

export const Availability = z.enum(["Standard", "Optional", "NotAvailable"]);
export const Powertrain = z.enum(["BEV", "PHEV", "HEV"]);
export const Drivetrain = z.enum(["FWD", "RWD", "AWD"]);
export const BodyStyle = z.enum(["SUV","Sedan","Truck","Hatchback","Minivan","Coupe","Wagon","Crossover","Van"]);
export const ChargePort = z.enum(["NACS","CCS","J1772","CHAdeMO"]);

export const Adas = z.object({
  systemName: z.string().nullable(),
  adaptiveCruiseControl: Availability,
  laneKeepAssist: Availability,
  laneCentering: Availability,
  handsFreeHighway: Availability,
  automaticEmergencyBraking: Availability,
  blindSpotMonitoring: Availability,
  selfParking: Availability,
  autonomyLevel: z.string(),           // "SAE L2", "SAE L2+", ...
});

export const Source = z.object({
  field: z.string().optional(),
  url: z.string().url(),
  asOf: z.string(),                    // ISO date
});

export const Vehicle = z.object({
  id: z.string().regex(/^[a-z0-9-]+$/), // slug
  make: z.string(), model: z.string(),
  year: z.number().int().min(2024).max(2026),
  trimsSummary: z.string(),
  bodyStyle: BodyStyle, powertrainType: Powertrain, segment: z.string(),
  electricRangeMi: z.number().nullable(),
  totalRangeMi: z.number().nullable(),
  mpge: z.number().nullable(), mpgCombined: z.number().nullable(),
  efficiencyMiPerKwh: z.number().nullable(),
  batteryKwhUsable: z.number().nullable(), batteryKwhTotal: z.number().nullable(),
  fuelTankGal: z.number().nullable(),
  dcFastMaxKw: z.number().nullable(), dcFast10to80Min: z.number().nullable(),
  acOnboardKw: z.number().nullable(), chargePort: ChargePort.nullable(),
  drivetrain: Drivetrain, motorLayout: z.string().nullable(),
  horsepower: z.number().nullable(), torqueLbFt: z.number().nullable(),
  zeroToSixtySec: z.number().nullable(), topSpeedMph: z.number().nullable(),
  towingCapacityLb: z.number().nullable(),
  adas: Adas,
  lengthIn: z.number().nullable(), widthIn: z.number().nullable(),
  heightIn: z.number().nullable(), wheelbaseIn: z.number().nullable(),
  groundClearanceIn: z.number().nullable(),
  cargoCuFt: z.number().nullable(), frunkCuFt: z.number().nullable(),
  seatingCapacity: z.number().nullable(), curbWeightLb: z.number().nullable(),
  msrpBaseUsd: z.number().nullable(), msrpTopUsd: z.number().nullable(),
  federalTaxCreditEligible: z.boolean().nullable(),
  federalTaxCreditNote: z.string().nullable(),
  warrantyBasic: z.string().nullable(), warrantyPowertrain: z.string().nullable(),
  warrantyBattery: z.string().nullable(),
  buildAndPriceUrl: z.string().url().nullable(),
  manufacturerUrl: z.string().url().nullable(),
  sources: z.array(Source).min(1),
  dataAsOf: z.string(),
}).superRefine((v, ctx) => {
  // powertrain-appropriate presence
  if (v.powertrainType === "BEV") {
    if (v.batteryKwhUsable == null && v.batteryKwhTotal == null)
      ctx.addIssue({ code:"custom", message:"BEV needs battery kWh", path:["batteryKwhUsable"] });
    if (v.electricRangeMi == null)
      ctx.addIssue({ code:"custom", message:"BEV needs electricRangeMi", path:["electricRangeMi"] });
  }
  if (v.powertrainType === "HEV") {
    if (v.fuelTankGal == null)
      ctx.addIssue({ code:"custom", message:"HEV needs fuelTankGal", path:["fuelTankGal"] });
    if (v.mpgCombined == null)
      ctx.addIssue({ code:"custom", message:"HEV needs mpgCombined", path:["mpgCombined"] });
  }
  if (v.powertrainType === "PHEV") {
    if (v.electricRangeMi == null)
      ctx.addIssue({ code:"custom", message:"PHEV needs electricRangeMi", path:["electricRangeMi"] });
    if (v.fuelTankGal == null)
      ctx.addIssue({ code:"custom", message:"PHEV needs fuelTankGal", path:["fuelTankGal"] });
  }
});
export type Vehicle = z.infer<typeof Vehicle>;
export type AvailabilityT = z.infer<typeof Availability>;
```

Sanity-range test (data-integrity, WS1) asserts, for non-null values:
`electricRangeMi 10–520`, `msrpBaseUsd 15000–250000`, `zeroToSixtySec 1.5–15`,
`seatingCapacity 2–9`, `year ∈ {2024,2025,2026}`, ids unique, ≥1 model per required brand.

### `shared/src/filters.ts` — query + DTO contract

```ts
export interface CarQuery {
  q?: string;
  powertrain?: ("BEV"|"PHEV"|"HEV")[];
  make?: string[];
  bodyStyle?: string[];
  drivetrain?: ("FWD"|"RWD"|"AWD")[];
  chargePort?: string[];
  minPrice?: number; maxPrice?: number;
  minElectricRange?: number;
  minSeating?: number;
  taxCreditOnly?: boolean;
  needsAdaptiveCruise?: boolean;   // ACC != NotAvailable
  needsLaneAssist?: boolean;       // laneKeepAssist != NotAvailable
  needsHandsFree?: boolean;        // handsFreeHighway != NotAvailable
  sort?: "price"|"range"|"zeroToSixty"|"efficiency"|"horsepower";
  order?: "asc"|"desc";
  page?: number; pageSize?: number;
}
export interface CarListResult { items: Vehicle[]; total: number; page: number; pageSize: number; facets: Facets; }
export interface Facets { makes: string[]; bodyStyles: string[]; }
```

### API contract (server ↔ web)

| Method | Path | Auth | Body / Query | Returns |
|---|---|---|---|---|
| GET | `/api/health` | – | – | `{ok:true, cars:number}` |
| GET | `/api/cars` | – | `CarQuery` (querystring) | `CarListResult` |
| GET | `/api/cars/:id` | – | – | `Vehicle` \| 404 |
| GET | `/api/compare?ids=a,b,c` | – | – | `{items: Vehicle[]}` (≤4) |
| POST | `/api/auth/register` | – | `{email,password}` | `{user:{id,email}}` + Set-Cookie |
| POST | `/api/auth/login` | – | `{email,password}` | `{user:{id,email}}` + Set-Cookie |
| POST | `/api/auth/logout` | – | – | `{ok:true}` clears cookie |
| GET | `/api/auth/me` | cookie | – | `{user}` \| 401 |
| GET | `/api/favorites` | cookie | – | `{carIds:string[]}` |
| POST | `/api/favorites` | cookie | `{carId}` | `{carIds}` |
| DELETE | `/api/favorites/:carId` | cookie | – | `{carIds}` |
| GET | `/api/cart` | cookie | – | `{carIds:string[]}` |
| POST | `/api/cart` | cookie | `{carId}` | `{carIds}` |
| DELETE | `/api/cart/:carId` | cookie | – | `{carIds}` |
| GET | `/api/export/cart?format=csv\|pdf` | cookie | – | file download |

Cookie: `token`, httpOnly, SameSite=Lax, Secure in prod, 7-day expiry. Auth errors → 401 JSON `{error}`. Validation errors → 400 JSON `{error, issues}`. `carId` must exist in the car store or → 404.

### `prisma/schema.prisma`

```prisma
datasource db { provider = "sqlite"; url = env("DATABASE_URL") }
generator client { provider = "prisma-client-js" }

model User {
  id        String   @id @default(cuid())
  email     String   @unique
  password  String
  createdAt DateTime @default(now())
  favorites Favorite[]
  cart      CartItem[]
}

model CartItem { id String @id @default(cuid()); userId String; carId String; addedAt DateTime @default(now()); user User @relation(fields:[userId], references:[id], onDelete: Cascade); @@unique([userId, carId]) }
```

## Execution: parallel workstreams

WS0 (scaffold) runs first and alone. Then WS1 (data), WS2 (backend), WS3 (frontend) run **in parallel** — they share only the frozen contracts above, no code dependency at build time (web uses MSW/mocked fetch in tests; server uses in-memory store; data validated against `shared`). WS4 (packaging) needs WS0. WS5 (debug/audit) runs last, after WS1–WS4 land.

### WS1 — Data curation fleet (parallel research agents)

**Task 1.1 — Freeze the schema.** Implement `shared/src/vehicle.ts`, `filters.ts`, `index.ts` exactly as the contracts above. Unit test: a hand-written valid BEV, PHEV, HEV each `.parse()` clean; a BEV missing battery fails; a HEV missing tank fails.

**Task 1.2 — Curation agents (fan-out, one per brand-group).** Each agent is given: the `Vehicle` Zod schema, the brand list slice, MY2024–2026, and returns a JSON array of schema-valid vehicles for its brands, every field sourced with `asOf` dates, `buildAndPriceUrl`/`manufacturerUrl` set to the real maker pages. `[Tesla]` · `[Ford, Lincoln]` · `[Chevrolet, GMC, Buick]` · `[Cadillac]` · `[Hyundai, Genesis]` · `[Kia]` · `[Toyota, Lexus]` · `[Honda, Acura]` · `[Nissan, INFINITI, Mitsubishi]` · `[Rivian, Lucid, VinFast]` · `[BMW, MINI]` · `[Mercedes-Benz]` · `[Audi, Volkswagen, Porsche]` · `[Volvo, Polestar]` · `[Subaru, Mazda]` · `[Jeep, Dodge, Ram, Chrysler, Fiat]` · `[Jaguar, Land Rover]` · `[Alfa Romeo, Maserati]`.

**Task 1.3 — Merge & validate.** Concatenate agent outputs → `data/cars.json`. Steps (TDD): write `data.test.ts` first (expects file + all-valid + brand coverage) → run (fails, no data) → curation fleet produces data → run (passes).
